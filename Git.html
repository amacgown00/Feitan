Git

1. Check current status


                git status
  • Shows new, modified, and deleted files.
  • Tells you if the working tree is clean.

2. Create folders


            mkdir -p 01-folder
  • -p makes parent folders if missing.

3. Create / overwrite a file

touch 01-folder/doc.md

Or write content directly from the terminal:

echo 'Same as it ever was' > 01-folder/doc.md

4. Delete files

rm 01-folder/doc.md

Or delete multiple files at once:

rm 01-folder/*.tmp

5. Stage changes (including deletes)

git add .

Stages all new, modified, or deleted files.
Optional: stage only tracked changes and deletions:

git add -u

6. Commit

git commit -m "Notes you can write about the change"

7. Pull remote changes

git pull --no--rebase

Merges remote changes into your local branch.
Finalize merge:

git commit

8. Push changes to GitHub

git push

Updates the remote repo to match your local
Deletes removed files on GitHub.

9. Quick Verify

git status
        git log --oneline -5

- git status – should say working tree clean.
git log --oneline — shows last 5 commits.

10. Undo mistakes safely

Undo unstaged changes

git restore FILE

Undo staged changes

git restore --staged FILE

Revert last commit (keep changes in files) 
git reset --soft HEAD-1
